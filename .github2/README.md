# Option 2: Prompt Files Pipeline

Build any application step-by-step using 8 sequential `.prompt.md` files. Each prompt generates artifacts that the next prompt builds on. The prompts are **app-agnostic** — you supply your own spec file and the prompts do the rest.

## Prerequisites

- VS Code with GitHub Copilot Chat extension
- Copilot Chat set to **Agent** mode (not Ask or Edit)
- An application specification markdown file in the repo root (e.g., `Survey-App.md`, `Todo-App.md`, etc.)

## How It Works

The `prompts/` folder contains 8 numbered prompt files, each handling one phase of development. You **attach your app spec file** when invoking each prompt — the prompts themselves contain no app-specific details.

| # | Prompt File | What It Does | Output |
|---|------------|-------------|--------|
| 1 | `01-architecture.prompt.md` | Tech stack, system design, folder structure | `docs/architecture.md` |
| 2 | `02-data-model.prompt.md` | Entities, ERD, database schema | `docs/data-model.md` |
| 3 | `03-api-design.prompt.md` | API contracts, OpenAPI spec | `docs/api-spec.md`, `docs/openapi.yaml` |
| 4 | `04-scaffold.prompt.md` | Project skeleton, configs, dependencies | Backend + frontend directories |
| 5 | `05-implement-backend.prompt.md` | Routes, services, auth, middleware | Working backend |
| 6 | `06-implement-frontend.prompt.md` | Pages, components, state management | Working frontend |
| 7 | `07-testing.prompt.md` | Unit, integration, and e2e tests | Test files alongside source |
| 8 | `08-documentation.prompt.md` | README, deployment guide, user guide | `README.md`, `docs/`, `CONTRIBUTING.md` |

## Steps

### Step 1: Add Your App Spec

Place your application specification markdown file in the repo root. This file should describe:
- What the app does
- User roles and their capabilities
- User stories or feature requirements
- Any constraints or bonus features

Example: `Survey-App.md`, `Kanban-Board.md`, `Recipe-App.md`, etc.

### Step 2: Open Copilot Chat in Agent Mode

1. Open the Copilot Chat panel in VS Code (`Ctrl+Shift+I`)
2. Switch to **Agent** mode using the mode picker at the top of the chat panel

### Step 3: Run Each Prompt in Order

Run the prompts **one at a time, in sequence**. Always attach your spec file using `#file:`.

**Phase 1 — Architecture:**
1. Type `/` in the chat input to open the prompt picker
2. Select `01-architecture`
3. Append your spec file: `#file:Your-App-Spec.md`
4. Press Enter to run it
5. Wait for Copilot to finish generating `docs/architecture.md`
6. **Review the output** — make sure the tech stack and structure look right before continuing

**Phase 2 — Data Model:**
1. Type `/02-data-model #file:Your-App-Spec.md` → Enter
2. Wait for `docs/data-model.md` to be generated
3. Review the ERD and entity definitions

**Phase 3 — API Design:**
1. Type `/03-api-design #file:Your-App-Spec.md` → Enter
2. Wait for `docs/api-spec.md` and `docs/openapi.yaml`
3. Review the endpoint definitions

**Phase 4 — Scaffold:**
1. Type `/04-scaffold #file:Your-App-Spec.md` → Enter
2. Wait for the backend and frontend directory trees to be created
3. Verify the folder structure matches the architecture doc

**Phase 5 — Backend:**
1. Type `/05-implement-backend #file:Your-App-Spec.md` → Enter
2. This is the longest phase — Copilot will create routes, services, middleware, and seed data
3. After completion, verify the server starts by installing dependencies and running the dev script

**Phase 6 — Frontend:**
1. Type `/06-implement-frontend #file:Your-App-Spec.md` → Enter
2. Copilot will create all UI components, pages, and API client
3. After completion, verify the app renders by installing dependencies and running the dev script

**Phase 7 — Testing:**
1. Type `/07-testing #file:Your-App-Spec.md` → Enter
2. Copilot generates test files alongside the source code
3. Run the test scripts to verify

**Phase 8 — Documentation:**
1. Type `/08-documentation #file:Your-App-Spec.md` → Enter
2. Copilot updates the README, creates deployment and user guides

### Step 4: Verify the Full App

After all 8 phases are complete, follow the quick start instructions in the generated `README.md` to install dependencies, set up the database, and start the application.

## Tips

- **Always attach your spec file.** Every prompt expects your app spec as input via `#file:`. Without it, the prompts have no requirements to work from.
- **Don't skip phases.** Each prompt references docs generated by earlier phases. Running them out of order will produce incomplete results.
- **Review after each phase.** If something looks off, ask Copilot to fix it before moving on. It's much easier to correct course early.
- **Start a new chat if context gets too long.** If Copilot responses degrade late in the pipeline, start a fresh chat session. The prompt files reference on-disk docs, so context is preserved.
- **You can re-run a single phase.** If phase 5 output isn't right, just re-run `/05-implement-backend #file:Your-App-Spec.md` — it reads from docs on disk, not chat history.
- **Works with any app spec.** These prompts are not tied to any specific application. Swap in a different spec file to build a completely different project.
